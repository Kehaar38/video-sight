## 確定仕様（現時点）

### 1. 目的
- HUD光学を使わず、`カメラ + ディスプレイ` で照準表示する「VideoSight」を開発する。
- HUD SIGHT本流の前段として、要素技術を分離して開発する。

### 2. ハード構成
- メインMCU: `XIAO ESP32S3 Sense`
- ディスプレイ: `Waveshare 1.69" 240x280`, `ST7789V2`
- IMU: `BNO055 (I2C, 0x28)`
- 入力: `InputDevice系（別MCU）を内蔵化`, I2C接続（現行アドレス `0x12`）
- バッテリー: `1セル18650` 方針
- 電源レール: `3.3V統一`

### 3. UI/操作
- エンコーダーは`距離入力専用`
- メイン操作は`4ボタン方針`（CENTER廃止の流れ）
- ボタン命名は最終的に `FRONT / BACK / UP / DOWN`
- `FRONT長押し=REC開始`, `FRONT短押し=SNAP`
- `BACK` は戻る/キャンセル系、`BACK長押し`でメニュー離脱
- 録画中も輝度変更可能（BRT変更は録画機能に非干渉）
- 押下判定は「`リリース時に短押し/長押し判定`」「長押し成立時は短押しを発火させない」

### 4. 電源・スリープ
- DeepSleep運用を前提
- `D0` をWake用途に使う方針（短押しで起床、長押しスリープ希望）
- 物理電源断用スイッチも併用する想定
- 低電圧時は分圧監視してDeepSleepへ移行する想定

### 5. ストレージ/ログ
- SDは「採用したいが必須ではない」
- 設定保存はNVS志向
- ログはシリアル中心、SD有効時は併用志向

## 準確定（設計方針はあるが実装未確定）

### 1. ピンアサイン
- D0〜D12含む暫定表あり（SD_CSは`GPIO21`想定）
- Shield/SPI競合対策として `JP1/JP2カット` を前提にする案あり
- D7/GPIO44は将来拡張用の予約候補

### 2. SDとLCDの共存
- SPI共存は実機検証前提
- 成立しなければ配線・構成をB案に切替える前提

### 3. メニュー構造
- List/Editの2層中心だが、階層深さは可変にしたい意向
